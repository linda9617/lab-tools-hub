Exp-Design Package Validator

Drop an exp-design ZIP package and get a single-page validation report: file inventory, metadata QC, config.json vs metadata cross-check, and metadata vs Echo picklist cross-check — every parameter scored PASS / WARN / ERROR before the experiment goes to the bench.

Run on GitHub

1How it works

Step 1

Drop ZIP & run validation

Drag in the exp-design ZIP package, click Run validation; every contained file is listed and parsed in one pass.

Step 2

Metadata QC

Plate names, regional conditions, sample types, condition columns and per-group replicate counts are summarized; structural checks (duplicate plate+well, bad well format, out-of-range wells, row/col mismatch) flagged as MATCH or FAIL.

Step 3

Config vs Metadata cross-check

Parameters from config.json (perturbagen list, regional conditions, concentrations, replicate count, plate count) are matched row-by-row against the metadata, with status flags per field.

Step 4

Metadata vs Echo picklist cross-check

Each picklist (perturbagen / treatment) is verified against active metadata rows: ID coverage, destination-well match, concentration / value match, transfer volume, source plate availability and dead-volume sufficiency. PDF report exports in one click.

2Preview

One drop, one verdict

Drop the exp-design ZIP and click Run validation: a top-level PASS / WARN / ERROR banner with totals (62 PASS, 0 WARN, 0 ERROR), config and design-summary parse status, run timestamps and the full file inventory of the package — all on a single screen, with a one-click Download report PDF.

Metadata at a glance

Active vs masked vs blank rows, plate names, regional conditions, sample types, condition columns and per-group replicate counts — all summarized from the metadata CSV. Structural checks (duplicate plate+well, bad well format, out-of-range wells, row/col mismatch) are scored MATCH / FAIL in the same panel.

Config vs metadata cross-check

Setup, perturbagen, randomization and condition regions parsed from config.json are laid out next to the active metadata, and a Cross-Check table flags every parameter (plate count, perturbagen list, regional conditions, concentrations, replicate count) as MATCH or mismatch — catching the silent design drift before instrument time.

Picklist cross-check, picklist by picklist

Every Echo picklist is verified twice: against active metadata rows (ID found, destination-well match, concentration / value match, transfer volume) and against source-plate availability (in source, source plate & well, sum transfer vs declared and usable volume after dead volume). Volume sufficiency is flagged per perturbagen, so dead-volume surprises don’t happen on the deck.
